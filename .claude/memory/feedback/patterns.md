# 検証済みのアプローチと成功パターン

## 設計パターン

- **コンテキストエンジニアリングの 5層モデルは有効**
  - **検証:** ADR、docs/、メモリシステムの分離により、情報が重複なく整理される
  - **推奨:** 新しい決定や学習をこのモデルに従って記録；層の境界を守る

- **ADR 形式での意思決定記録**
  - **検証:** ADR-001～005 の決定により、根拠が明確で再検討が最小化される
  - **推奨:** 建築的重要性を持つ決定は常に ADR として記録

## ドキュメント構造

- **docs/ 配下への統一は保守性を向上させる**
  - **検証:** DESIGN.md vs docs/design.md の重複解消で、管理が単純化
  - **推奨:** ドキュメントは docs/ 配下に集約；重複を避ける

- **メモリシステムでセッション横断的知識を保持**
  - **検証:** 過去セッションの学習をメモリに記録することで、同じ間違いを繰り返さない
  - **推奨:** 非自明な学習は毎セッション更新；古い情報は削除

## 開発ワークフロー

- **機能実装前に EnterPlanMode で設計確認**
  - **検証:** 計画なしで実装すると、後から大きなリファクタリングが必要になる
  - **推奨:** 複雑な実装タスクは常に計画；小さなタスクはスキップ可能

- **TaskCreate でタスクを細分化**
  - **検証:** 進捗追跡が明確になり、完了感が高まる
  - **推奨:** 非自明な作業はタスクとして管理
