# ADR-003: Signal/Noise デザインシステム採用

## Status
Accepted (2026-05-20)

## Context

ポートフォリオの視覚的デザインを定義する際、統一されたデザイン言語が必要でした。

**課題**:
- 複数ページで色、サイズ、間隔が一貫性を欠く
- デザイントークン（色、タイポ、スペーシング）の定義が不明確
- 重要情報と背景情報の視覚的階層が曖昧

**選択肢**:
1. Material Design（Google）
2. iOS Human Interface Guidelines（Apple）
3. Signal/Noise（カスタム）

## Decision

**Signal/Noise デザインシステムを採用。**

### 核心概念

- **Signal（信号）**: ユーザーに重要な情報を強調する（色、サイズ、太字）
- **Noise（ノイズ）**: 不要な視覚的要素を最小化する（グレースケール、透明度、軽い重さ）

### 実装

- **色**: Primary（Signal）と Neutral（Noise）の 2 色系
- **タイポ**: 見出し（700 weight）vs 本文（400 weight）で階層を表現
- **スペーシング**: 8px ベースの標準スケール

## Consequences

### ✅ メリット

- **シンプルで統一的**: 色数が少なく、判断が単純化
- **アクセシビリティ**: Noise（グレー）を背景化することで、対比が明確
- **拡張性**: Signal/Noise の 2 軸で新しい配色が自動生成可能
- **ブランド統一**: ポートフォリオ全体として一貫したビジュアル

### ❌ デメリット

- **視覚的多様性の制約**: 色が限定される（カラフルさが減少）
- **トークン定義の手間**: 詳細な色・フォント・間隔を事前に定義が必要
- **既存デザイン資産の非互換**: Material Design など標準的な UI kit との互換性が低い

## Rationale

ポートフォリオは「個人ブランドの表現」であり、一般的な Material Design より「独自性」と「シンプルさ」を優先すべき。

Signal/Noise により、シンプルかつ高度なビジュアル表現が可能。

## Alternatives Considered

### 1. Material Design 3（採用せず）

**メリット**: 
- 豊富な UI component サンプル
- 大規模な企業アプリで採用実績

**デメリット**: 
- テンプレート的で個性が失われやすい
- ポートフォリオには「汎用性」より「個性」が必要

**決定理由**: ポートフォリオのユースケースと合致しない。

### 2. Fully Custom（全く独自）（採用せず）

**メリット**: 完全な自由度

**デメリット**: 
- デザイン判断が個人化し、チームスケール時に再定義が必要
- デザインガイドラインが不在で管理が煩雑

**決定理由**: 最小限のシステムを定義することで、スケーラビリティを確保。
