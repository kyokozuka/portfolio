## TypeScript strict モード

**決定**: TypeScript を `strict: true` で採用し、すべてのコードに明示的な型定義を必須とする。

**理由**: 型安全性による実行時エラー削減と、型定義自体が実装ドキュメントになる保守性を優先する。

---

## Next.js App Router 流儀アーキテクチャ

**決定**: フレームワーク非依存の FSD ではなく、Next.js App Router の流儀に沿った構造を採用する。

- **ロケールルーティング**: `app/[lang]/`（`/en` `/ja`）。`lib/i18n` の `locales`/`isLocale`/`getDictionary` を単一ソースとし、`generateStaticParams` で静的書き出し（`output: 'export'`）する。ルート `/` はデフォルトロケールへリダイレクト。
- **レンダリング**: Server Component 既定。`'use client'` はインタラクティブな葉（言語切替・`useReveal` を使う節など）に限定する。
- **コロケーション**: ルート専用コンポーネントは当該ルート配下（`app/**/_components/`）に置き、横断利用が生じた時点で `components/` へ昇格する。
- **重複ルートの集約**: 同型の詳細ページは `[slug]` 動的ルートに集約する。設計が大きく異なる場合は slug→コンポーネントのレジストリで分岐する。
- **ディレクトリ**: `app/`（ルート）・`components/`（横断 UI）・`lib/`（i18n・hooks 等）・`content/`（型付きコンテンツ）。`shared/` は使わない。
- **インポート方向**: `app → components → lib`。上方向・逆流 import を禁止する。

**理由**: 静的ポートフォリオの規模には FSD の抽象は過剰で、依存方向違反・i18n 破綻・SSOT 分裂を招いていた。Next.js 標準に沿うことでフレームワーク機能（RSC・ロケールルーティング）を活かし、保守性を高める。

**残作業**: `features/*` の UI セクション・data・types は移行途中で残存しており、`app/**/_components/`・`content/` への物理移設は後続作業。

## Signal/Noise デザインシステム

**決定**: Signal/Noise デザインシステムを採用する。

- **Signal（信号）**: 重要情報を色・サイズ・太字で強調する。
- **Noise（ノイズ）**: 不要な視覚要素をグレースケール・透明度・軽いウェイトで抑制する。
- **実装**: Primary（Signal）/ Neutral（Noise）の2色系、見出し 700・本文 400 のウェイト階層、8px ベースのスペーシング。

詳細は [design.md](./design.md) を参照。

## セマンティックHTML必須

**決定**: ジェネリックな `<div>` + CSS クラスではなく、意図を反映したセマンティック HTML 要素を使用する。

### 要素マッピング

| 用途 | 使う要素 |
| ---- | -------- |
| ページ見出し・ナビ・フッター | `<header>` / `<nav>` / `<footer>` |
| メインコンテンツ | `<main>` |
| セクション | `<section aria-labelledby="...">` |
| 自己完結コンテンツ | `<article>` |
| リスト | `<ul>` + `<li>` |
| 補足情報 | `<aside>` |
| 連絡先 | `<address>` |

### `<div>` / `<span>` の許容用途

`<div>` / `<span>` は「非モダン」ではなく、**意味を持たない純粋なレイアウト・装飾のまとまり**には正しく使う。原則は次の一文に集約する。

> **意味があればセマンティック要素、意味がなくスタイル目的だけなら `<div>` / `<span>`。**

- ✅ 許容: グリッド/フレックスのラッパー、余白・境界のボックス、アニメーション用ラッパー、インライン装飾スパン。
- ❌ 非許容: `section`/`article`/`nav` 等で表せる箇所を `<div>` で代替すること（div スープ）。
- 💡 不要な中間ラッパーは **React Fragment（`<>...</>`）** に置換し、要素数を増やさない。

**理由**: アクセシビリティ（スクリーンリーダー）・SEO・保守性はポートフォリオのブランド価値に直結する。WCAG 準拠を満たす。

## Tailwind CSS

**決定**: Tailwind CSS（v3+）をユーティリティファーストで採用する。

- **プリセット**: 色・フォント・スペーシングを設定で統一する。
- **抽象化**: `@apply` はコンポーネントレベルの最小限に留める。
- **レスポンシブ / Dark Mode**: `sm:`/`md:`/`lg:`・`dark:` プレフィックスで対応する。

**理由**: デザイントークンを一貫管理でき、迅速な開発と保守性を両立する。
